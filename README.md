## 1. Two Sum (兩數之和)

### 核心概念
* **資料結構**：`Hash Table` (Python `dict`)
* **時間複雜度**：$O(n)$

### 關鍵問題 Q&A
**Q：為什麼不事先把所有值都放進 `pair_idx`，而是「邊找邊放」？**
1. **處理重複數值**：
   - 若陣列為 `[3, 3]` 且目標為 `6`。若先全部放入，第二個 `3` 的索引會覆蓋掉第一個。
   - 「邊找邊放」能在讀取到第二個 `3` 時，於字典中找到第一個 `3` 的索引，完美解決衝突。
2. **提前結束 (Early Exit)**：
   - 若答案出現在陣列的前端，程式不需要掃描完整個陣列，能提升執行效率。
3. **防止自我匹配**：
   - 確保不會找到自己（例如 $target - num = num$ 的情況）。

## 2. Valid Parentheses (有效的括號)

### 核心概念
* **資料結構**：`Stack` (堆疊，後進先出 LIFO)
* **時間複雜度**：$O(n)$

### 語法要點
* **`not stack`**：
  - 在 Python 中，空列表、空字串會被視為 `False`。
  - `not stack` 為 `True` 代表堆疊是空的。用於檢查「是否有剩餘左括號」或「右括號是否多出」。
* **`stack.pop()`**：
  - **取出並刪除**：回傳列表最後一個元素，並同時將其從原列表中移除。

### 關鍵問題 Q&A
**Q：為什麼 `elif char in mapping:` 不直接使用 `else:`？**
1. **排除非法字元 (Robustness)**：
   - 明確過濾右括號。若字串中混入空格或其他字元，`elif` 寫法能確保這些字元不被誤當作右括號處理，提升程式的健壯性。
2. **可讀性 (Readability)**：
   - 程式碼具有自解釋性，讓讀者能直觀地區分「處理左括號」與「處理右括號」的邏輯區塊。

| 語法 | 說明 |
| :--- | :--- |
| `-> List[int]` | **回傳值提示**：說明函式執行完會回傳一個由整數組成的列表。 |
| `enumerate()` | **枚舉**：同時獲取元素的「索引(Index)」與「數值(Value)」。 |
| `dict.values()` | **字典值清單**：用於檢查某個元素是否為字典中的任何一個 Value。 |
